🔍 시나리오 1: 정상 결제 플로우 테스트

1. 페이지 로드 및 UI 검증
- [ ] 결제 페이지가 정상적으로 로드되는지 확인
- [ ] 모든 입력 필드가 올바르게 표시되는지 확인
- [ ] 애니메이션 효과가 부드럽게 작동하는지 확인
- [ ] 제목 "💳 결제 페이지"가 표시되는지 확인

2. 카드 정보 입력 테스트
- [ ] 카드 번호 필드에 "1234567890123456" 입력
- [ ] 자동으로 "1234 5678 9012 3456" 형식으로 포맷팅되는지 확인
- [ ] 카드 소유자 필드에 "홍길동" 입력
- [ ] 만료일 필드에 "12/25" 입력
- [ ] CVV 필드에 "123" 입력
- [ ] 결제 금액 필드에 "50000" 입력
- [ ] 결제 내역 필드에 "온라인 쇼핑몰 결제" 입력

3. 결제 처리 테스트
- [ ] "결제하기" 버튼 클릭
- [ ] "결제 처리 중..." 메시지가 표시되는지 확인
- [ ] 2초 후 성공 메시지 "✅ 결제가 성공적으로 완료되었습니다!" 표시 확인
- [ ] 성공 후 모든 입력 필드가 초기화되는지 확인

❌ 시나리오 2: 폼 검증 테스트

1. 필수 필드 검증
- [ ] 모든 필드를 비우고 "결제하기" 버튼 클릭
- [ ] "모든 필드를 입력해주세요" alert 메시지가 표시되는지 확인
- [ ] alert 확인 후 폼이 초기 상태로 유지되는지 확인

2. 카드 번호 형식 검증
- [ ] 카드 번호 필드에 "123456789012345" (15자리) 입력
- [ ] "결제하기" 버튼 클릭
- [ ] "올바른 카드 번호를 입력해주세요" alert 메시지가 표시되는지 확인

3. CVV 형식 검증
- [ ] CVV 필드에 "12" (2자리) 입력
- [ ] "결제하기" 버튼 클릭
- [ ] "올바른 CVV를 입력해주세요" alert 메시지가 표시되는지 확인

4. 결제 금액 검증
- [ ] 결제 금액 필드에 "0" 입력
- [ ] "결제하기" 버튼 클릭
- [ ] "올바른 금액을 입력해주세요" alert 메시지가 표시되는지 확인

🔒 시나리오 3: 입력 형식 및 특수 문자 테스트

1. 카드 번호 자동 필터링

- [ ] 숫자가 아닌 문자가 제거되는지 확인
